## ⚙️ การตั้งค่า Environment Variables

สร้างไฟล์ `.env` ในโฟลเดอร์หลัก (Root Directory) ของโปรเจกต์ และใส่ค่าการเชื่อมต่อดังนี้:

```env
# URL สำหรับติดต่อ API ฝั่ง Client (รันเซิร์ฟเวอร์เดี่ยวบนพอร์ต 3001)
NEXT_PUBLIC_API_URL=http://localhost:3001

# JWT Secret Key สำหรับการเข้ารหัสสิทธิ์ผู้ใช้งาน (สุ่มคีย์ยาวๆ ที่ปลอดภัย)
JWT_SECRET=your_jwt_secret_key_here

# MySQL / TiDB Cloud Database Configuration
DB_HOST=gateway01.ap-southeast-1.prod.alicloud.tidbcloud.com
DB_USER=your_db_username
DB_PASSWORD=your_db_password
DB_NAME=swimming_pool_db
DB_PORT=4000
DB_SSL=true
DB_SSL_REJECT_UNAUTHORIZED=false

# Cloudinary Storage Configuration
CLOUDINARY_CLOUD_NAME=your_cloudinary_cloud_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret
```

## 🗄 การตั้งค่าฐานข้อมูลด้วย Drizzle ORM

ระบบใช้ Drizzle ORM ในการจัดการโครงสร้างและคำสั่งฐานข้อมูลแบบ Type-Safe:

### การตรวจสอบ Schema ฐานข้อมูลปัจจุบัน
*   ไฟล์ Schema หลักจะอยู่ที่ [lib/db/schema.ts](file:///d:/swimming-pool-system/swimming-pool-system/lib/db/schema.ts)

### การจัดการโครงสร้างผ่าน Drizzle Kit CLI
รันคำสั่งเหล่านี้จาก root ของโปรเจกต์ได้เลยครับ:

```bash
# 1. พุชโครงสร้าง Schema ล่าสุดในโค้ดขึ้นไปยัง Database โดยตรง
npx drizzle-kit push

# 2. ดึงโครงสร้างล่าสุดจาก Database เข้ามาสร้างเป็นโค้ดไฟล์ Schema ท้องถิ่น
npx drizzle-kit pull

# 3. เปิดใช้งาน Drizzle Studio (โปรแกรม GUI หน้าเว็บเพื่อเข้าไปดูและแก้ไขตารางข้อมูลโดยตรง)
npx drizzle-kit studio
```

## 🏃‍♂️ การรันระบบ

เนื่องจากเซิร์ฟเวอร์ถูกรวมเป็นหนึ่งเดียว (Unified Next.js) คุณไม่จำเป็นต้องแยกรัน backend กับ frontend อีกต่อไป:

### การรันในโหมด Development
```bash
npm run dev
```
*   **Frontend**: เข้าใช้งานผ่านเบราว์เซอร์ที่ **`http://localhost:3001`**
*   **Backend API**: ถูกรันโดยอัตโนมัติอยู่ภายใต้ **`http://localhost:3001/api`** (เช่น ตรวจสอบความพร้อมการทำงานผ่าน `/api/health`)

### การรันในโหมด Production
```bash
# Compile หน้าเว็บและ TypeScript
npm run build

# เริ่มรันระบบสำหรับ Production
npm run start
```

## 🛠 การแก้ไขปัญหา

### 1. ปัญหาการเชื่อมต่อ Database ค้างตอน Build โครงการ
*   **สาเหตุ**: การเปิด Pool การเชื่อมต่อโดยตรงทันทีเมื่อคอมไพล์ Next.js จะทำให้การตรวจสอบหน้า static ค้างระหว่างรัน `next build`
*   **ทางแก้**: ระบบปัจจุบันได้ห่อหุ้ม instance ฐานข้อมูลด้วย `Proxy` ใน [lib/db/index.ts](file:///d:/swimming-pool-system/swimming-pool-system/lib/db/index.ts) เพื่อรันการเชื่อมต่อแบบขี้เกียจ (Lazy Initialization) หากพบปัญหานี้ ให้เช็คว่าโค้ดไม่มีการเปิดเชื่อมต่อตรงภายนอก Proxy

### 2. ปัญหารูปสลิปไม่ขึ้น / อัปโหลดไม่เข้า
*   **ทางแก้**: ให้ตรวจสอบว่าตั้งค่าตัวแปร `CLOUDINARY_*` ใน `.env` ครบถ้วนและถูกต้อง รวมถึงตรวจสอบให้แน่ใจว่าเครื่องเซิร์ฟเวอร์สามารถคุยออกเน็ตเวิร์กสาธารณะเพื่ออัปโหลดเข้าเซิร์ฟเวอร์ Cloudinary ได้

## 📊 โครงสร้างตารางข้อมูล

ระบบฐานข้อมูลประกอบด้วย 12 ตารางหลัก:
1.  **`users`**: ข้อมูลบัญชีผู้ใช้งาน
2.  **`user_categories`**: ประเภทประเภทของบุคคล (นักเรียน/นิสิต/บุคลากร/บุคคลทั่วไป) และราคาค่าบริการ
3.  **`membership_types`**: ประเภทแพ็กเกจสมาชิก (รายวัน/รายปี)
4.  **`memberships`**: ข้อมูลประวัติการถือครองอายุสมาชิกภาพ
5.  **`pool_resources`**: ข้อมูลทรัพยากรสระว่ายน้ำ (สระหลัก/สระเด็ก) และสถานะ
6.  **`pool_schedules`**: เวลาเปิด-ปิดของแต่ละสระแยกรายวัน จันทร์ - อาทิตย์
7.  **`reservations`**: ประวัติและการบันทึกข้อมูลการจองใช้งานสระว่ายน้ำ
8.  **`payments`**: บันทึกธุรกรรมการเงิน ยอด ยืนยันสิทธิ์ และลิงก์สลิปคลาวด์
9.  **`lockers`**: รหัสล็อกเกอร์ โซนที่ตั้ง และสถานะ
10. **`locker_reservations`**: การบันทึกข้อมูลการจองตู้ล็อกเกอร์คู่กับการเข้าใช้งาน
11. **`system_settings`**: ข้อมูลการตั้งค่าส่วนกลางของสระว่ายน้ำ
12. **`notifications`**: ข้อมูลรายการแจ้งเตือนสำหรับผู้ใช้งาน
